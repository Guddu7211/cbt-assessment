Option 1 — Railway
Free · 500hrs/month · Auto-deploys from GitHub · No credit card needed
⭐ RECOMMENDED
What you need: A free GitHub account (github.com) and a free Railway account (railway.app). Both take 1 minute each.
Install Git (if not already installed)
Download from git-scm.com and install. Then open a terminal (Command Prompt / PowerShell on Windows, Terminal on Mac).
Create a GitHub repository
Go to github.com/new → name it cbt-assessment → click Create repository.
Push your files to GitHub
Open your terminal inside the folder where your files are, then run these commands one by one:
git init git add . git commit -m "Initial commit" git branch -M main git remote add origin https://github.com/YOUR_USERNAME/cbt-assessment.git git push -u origin main
Replace YOUR_USERNAME with your actual GitHub username.
Create a Railway account
Go to railway.app → click Start a New Project → sign up with your GitHub account (click "Login with GitHub").
Create a new Railway project
After logging in → click New Project → select Deploy from GitHub repo → select your cbt-assessment repo → click Deploy Now.
Add a PostgreSQL database
Inside your Railway project → click + New → select Database → select Add PostgreSQL. Railway will automatically connect it to your app via DATABASE_URL.
✅ The database tables are created automatically when the app starts — you don't need to do anything in the database.
Set environment variables
Click on your web service (not the database) in Railway → go to Variables tab → click Add Variable for each:
Variable NameValue
ADMIN_KEYChoose a strong password e.g. MyAdmin@2024
NODE_ENVproduction
DATABASE_URL is added automatically by Railway when you attach the PostgreSQL database. Do NOT add it manually.
Get your live URL
Click on your web service → go to Settings tab → scroll to Domains → click Generate Domain. You'll get a URL like:
https://cbt-assessment-production.up.railway.app
🎉 Your app is now live! Share /index.html with candidates and /admin.html for yourself.

Option 2 — Render
Free · App sleeps after 15min idle · 1GB PostgreSQL
FREE
Push to GitHub (same as Railway step 1–3)
Follow steps 1–3 from the Railway guide above to get your code on GitHub.
Sign up at Render
Go to render.com → click Get Started for Free → sign up with GitHub.
Create a PostgreSQL database
New + → PostgreSQL → Name: assessment-db → Plan: Free → click Create Database. Copy the External Database URL.
Create a Web Service
New + → Web Service → connect your GitHub repo → set:
FieldValue
Build Commandnpm install
Start Commandnode server.js
PlanFree
Add environment variables
In the web service → Environment tab → add:
NameValue
DATABASE_URLPaste the External Database URL from step 3
ADMIN_KEYYour chosen admin password
NODE_ENVproduction
Deploy
Click Create Web Service. Render will build and deploy automatically. Your URL will be shown at the top of the page.
Note: On the free Render plan, the app sleeps after 15 minutes of no traffic. It wakes up in ~30 seconds when someone visits. Consider Railway if this is a problem.

After Deployment — First Steps
Open your Admin Dashboard
Go to https://YOUR-URL/admin.html → log in with your ADMIN_KEY.
Import your Questions
Go to Questions → click ⬇ Questions Template to download the CSV template → fill it with your questions → click ⬆ Import CSV/Excel to upload.
Import your Candidates
Go to Candidates → download the template → fill with employee names, IDs and emails → import.
Configure Exam Settings
Go to Exam Control → set Total Time, Max Questions per candidate, Tab Switch limit → click Save Settings.
Start the Exam
Click ▶ Start Exam. Share https://YOUR-URL/index.html with candidates.
Monitor in real time
Go to Live Monitor → enable Auto every 10s to see who is active, their progress, remaining time, and tab switches.
Download Results
Go to Results → click ⬇ Export Excel to download a complete results spreadsheet.
🎉 You're all set! Your CBT system is live and ready for candidates.
